The Staplehire CLI (`staplehire`) runs the hiring loop from your terminal or editor — **source, pipeline, investigate, interview, hire** — with **JSON output** when piped and readable tables in an interactive shell.

```bash
npm install -g @staplehire/staplehire-cli
staplehire login
staplehire roles list --json
```

Latest release: **`@staplehire/staplehire-cli@0.7.1`** on [npm](https://www.npmjs.com/package/@staplehire/staplehire-cli).

## Domains

| Host | Purpose |
| --- | --- |
| [staplehire.com](https://staplehire.com) | Marketing site and these docs (`/docs`) |
| [app.staplehire.com](https://app.staplehire.com) | Product dashboard, OAuth `/connect`, apply links |
| [api.staplehire.com](https://api.staplehire.com) | Hiring API gateway (CLI default for `login` / production keys) |

API responses include a `urls` object on roles, candidates, and sessions — deep links point at **`app.staplehire.com`** (for example `urls.review_url` on a candidate with an interview session).

## What you can do

| Stage | CLI focus |
| --- | --- |
| **Source** | Create roles from a JD, start sourcing with search criteria, list prospects and run logs |
| **Pipeline** | List roles and candidates, move stages, read `urls` from JSON |
| **Investigate** | Enrich and research candidates, poll async jobs, read screening results |
| **Interview** | Create designs, send invites, list sessions (signed recording URLs in JSON) |
| **Hire** | Send agent email and advance candidates through stages |

## Output modes

- **Interactive terminal** — tables and prompts for humans.
- **Agents / scripts** — `--json` or pipe stdout for machine-readable records. Run `staplehire commands` for the full tree.
