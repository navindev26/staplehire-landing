`candidates enrich` starts an async **deep research** job on a candidate. The command returns immediately with a job ID — you must [poll the job](/poll-agent-jobs) until it's `done`, then read the result with `candidates research`.

```bash
JOB_ID=$(staplehire candidates enrich <candidateId> | jq -r '.job.id')
staplehire jobs poll "$JOB_ID"
staplehire candidates research <candidateId>
```

## The enrich → poll → read pattern

<Steps>
  <Step title="Start the job">
    ```bash
    JOB_ID=$(staplehire candidates enrich <candidateId> \
      --prompt "Focus on backend systems, TypeScript, and startup experience." \
      | jq -r '.job.id')
    ```

    ```json
    {
      "job": { "id": "job_…", "type": "enrich", "status": "pending", "candidate_id": "cand_…" },
      "created": true
    }
    ```
  </Step>
  <Step title="Poll until done">
    ```bash
    staplehire jobs poll "$JOB_ID"
    ```

    `jobs poll` blocks until the job reaches `done` or `failed`. Exit `0` means done; exit `9` means the poll timed out (the job may still be running).
  </Step>
  <Step title="Read the research">
    ```bash
    staplehire candidates research <candidateId>
    ```

    ```json
    {
      "research": {
        "summary": "Candidate has recent backend platform experience.",
        "highlights": ["Built TypeScript services", "Worked in early-stage startups"],
        "sources": ["https://linkedin.com/in/…"]
      }
    }
    ```
  </Step>
</Steps>

### Parameters

| Flag | Required | Description |
| --- | --- | --- |
| `<candidateId>` | Yes | Candidate to research |
| `--prompt <text>` | No | Extra research instructions to focus the job |
| `--force` | No | Start a fresh job even if one is already in flight |

## Handle in-flight jobs

If the response has `"created": false`, an enrich job is **already running** for this candidate. Don't start another — poll the returned `job.id`.

```bash
RESP=$(staplehire candidates enrich <candidateId>)
JOB_ID=$(echo "$RESP" | jq -r '.job.id')
CREATED=$(echo "$RESP" | jq -r '.created')
echo "created=$CREATED — polling $JOB_ID"
staplehire jobs poll "$JOB_ID"
```

Pass `--force` only when you intentionally want to re-run research:

```bash
staplehire candidates enrich <candidateId> --force
```

## One-liner

```bash
JOB_ID=$(staplehire candidates enrich <candidateId> | jq -r '.job.id') \
  && staplehire jobs poll "$JOB_ID" -q \
  && staplehire candidates research <candidateId> | jq -r '.research.summary'
```

## Read screening results

Separate from research, fit/screening results (available after an interview) are read with:

```bash
staplehire candidates screening-result <candidateId>
```

## Common errors

| Error | Exit code | Fix |
| --- | --- | --- |
| `NotFoundError` | `4` | Candidate does not exist — run `candidates list` or `candidates get` |
| Poll timeout | `9` | Increase `--timeout`, or inspect later with `jobs get <jobId>` |
| `job_failed` | `1` | Read `error.message`; fix inputs or retry with `--force` |
| `AuthenticationError` | `2` | Run `staplehire login` or set `STAPLEHIRE_KEY` |

## FAQ

<AccordionGroup>
  <Accordion title="Is enrichment synchronous?">
    No. It creates a job and returns immediately. Poll the job before calling `candidates research`.
  </Accordion>
  <Accordion title="Can I customize the research focus?">
    Yes. Pass `--prompt` to steer what the deep-research job investigates.
  </Accordion>
  <Accordion title="What does `\"created\": false` mean?">
    A research job for this candidate is already in flight. Poll the returned `job.id` instead of starting a duplicate. Use `--force` to deliberately re-run.
  </Accordion>
</AccordionGroup>
