## Install skills

```bash
staplehire skills install
```

Copies the **`staplehire`** agent skill from [`@staplehire/staplehire-cli`](https://github.com/staplehire/staplehire-cli) into supported editors. Agents load it before running `staplehire` so they use `--json`, poll async `jobs`, and read `urls` from responses instead of guessing IDs or hosts.

## Supported editors

| Editor | Setup |
| --- | --- |
| **Cursor** | `staplehire skills install`; run CLI in agent terminal |
| **Claude Code** | Skill + `staplehire commands` for discovery |
| **OpenAI Codex** | Same CLI and JSON contract as the dashboard |
| **OpenClaw** | Skills + messaging gateways |
| **Hermes Agent** | Long-running hiring workflows |
| **Pi** | Terminal harness + npm skills |

## Agent workflow

1. `staplehire login` (or `STAPLEHIRE_KEY` from [app.staplehire.com/settings/developers](https://app.staplehire.com/settings/developers))
2. `staplehire roles list --json`
3. Stage commands with `--json` (source → pipeline → investigate → interview → hire)
4. Open **`urls.review_url`**, **`urls.dashboard_url`**, etc. from JSON — they point at **app.staplehire.com**, not the marketing site

<Warning>
  Do not guess role or candidate IDs — list or create via the CLI first. Do not hardcode `staplehire.com/roles/…`; use `urls` from API responses.
</Warning>
