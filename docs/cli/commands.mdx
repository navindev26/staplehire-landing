Commands follow `staplehire <resource> <action>`. Add **`--json`** when piping output to an agent or script.

<Tip>
  Run `staplehire --help` or `staplehire <resource> --help` for the live command tree installed with your version.
</Tip>

## Source

Create a role from a job description, start sourcing, and list prospects:

```bash
staplehire login
staplehire roles create --jd @senior-sdr.md
staplehire sourcing start <roleId>
staplehire sourcing prospects <roleId> --json
```

## Agent inbox

Each role has a virtual inbox. List messages and triage applications:

```bash
staplehire roles list --json
staplehire inbox messages <roleId> --json
staplehire inbox triage <roleId> --surface-above 7.5
```

Role inboxes use addresses like `hire+<role>@in.staplehire.com` so applications land in the right pipeline.

## Investigate

Enrich a candidate, run research, and read screening results before you interview:

```bash
staplehire candidates enrich <candidateId>
staplehire candidates research <candidateId>
staplehire candidates screening-result <candidateId> --json
```

## Interview

Create an interview design, send an invite, and list sessions:

```bash
staplehire designs create <roleId> --brief "SDR screen"
staplehire candidates send-interview <candidateId> --design <designId>
staplehire sessions list <roleId> --json
```

## Hire

Preview and send agent mail from the terminal:

```bash
staplehire candidates send-email <candidateId> \
  --subject "Staplehire" \
  --title "Quick intro" \
  --body @message.md
```

## JSON output

When stdout is not a TTY or you pass `--json`, responses are structured records (roles, prospects, inbox messages, screening scores, sessions) that agents can parse without scraping the dashboard.

```bash
staplehire sourcing prospects rol_abc123 --json | jq '.prospects[] | select(.score >= 8)'
```
